## ?? Key Design Decisions & Notes

### 1. Authentication
| Demo | Production |
|------|------------|
| Cookie Auth (built-in) | ASP.NET Identity / JWT / OAuth2 |
| HMACSHA512 (salt+hash) | BCrypt / Argon2 |
| `[ValidateAntiForgeryToken]` trĻn form | + Rate limiting, Account lockout |
| 7-day sliding cookie | + 2FA, Email confirm, External login |

### 2. Real-time (SignalR)
| Demo | Production |
|------|------------|
| In-process, single server | Redis Backplane / Azure SignalR Service |
| WebSocket (auto-fallback) | + Message queue (RabbitMQ/Kafka) |
| Room-based Groups | + User-specific channels |
| Debounced typing indicator | + Presence channel |
| Auto-reconnect 5 attempts | + Exponential backoff |

### 3. Caching
| Demo | Production |
|------|------------|
| `IMemoryCache` (in-process) | Redis (`IDistributedCache`) |
| Cache-aside pattern | + Write-through cho hot data |
| Manual invalidation on write | + Event-driven invalidation (pub/sub) |
| Room list: 2min sliding / 10min absolute | Tune per traffic pattern |
| Online users: static Dictionary | Redis Hash + TTL + Pub/Sub |

### 4. Database
| Demo | Production |
|------|------------|
| `EnsureCreated()` | EF Migrations / SQL scripts |
| Composite index (RoomId+SentAt) | + Covering indexes, partitioning |
| Cursor pagination (beforeId) | ? Best for chat/feed |
| No soft delete | + IsDeleted, EditedAt fields |
| Single DB | + Read replicas, sharding |
